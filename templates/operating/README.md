# Operating Templates Proposal

These are proposed operating templates for review. They are intended to reduce daily setup cost without replacing human judgement.

## Proposed First Set

| Template | File | Purpose | Review question |
| --- | --- | --- | --- |
| Daily issue | `daily-issue.md` | Turn the daily GitHub issue into the build cockpit. | Does this include the right checklists without becoming busywork? |
| Evidence pack | `evidence-pack.md` | Make missing evidence visible before publication. | Is the threshold practical for compressed days? |
| Article / case study | `article-or-case-study.md` | Derive public content from the README and evidence. | Does it keep daily content short enough? |
| Social posts | `social-posts.md` | Draft concise posts that point back to evidence. | Does it avoid unsupported claims and hype? |
| Weekly review | `weekly-review.md` | Calibrate verdicts, confidence and repeated findings. | Does it capture synthesis without rewriting history? |

## Review Notes

- These templates are drafts.
- The foundation docs remain the authority.
- Update templates only when they reduce friction, improve evidence quality or clarify decisions.
